# OCR · AI첨삭 API 연동 설계서 — 키 확보 후 적용

> 현재 프로토타입은 두 모듈이 **목업**이다. 키가 생기면 이 두 함수만 교체하면 실서비스로 전환된다.
> - `ocrModule(image)` → 손글씨 이미지 → 텍스트
> - `aiReview(text, book)` → 텍스트 → {칭찬, 개선, 발전, 성취수준}
> ⚠️ 대회 규정: **학생이 생성형 AI를 직접 호출하면 연령제한 약관 위반 = 등외.** 모든 LLM/OCR 호출은 **서버측(교사 운영 키)**에서.

---

## 1. 아키텍처 (연령제한·개인정보 안전 구조)
```
[학생 브라우저] --이미지/텍스트--> [우리 서버(함수)] --키 보관--> [OCR API] / [LLM API]
                                         |
                                    학생은 키를 모름 / 직접 호출 안 함
```
- 키는 **서버 환경변수**에만. 클라이언트 노출 금지.
- Firebase 사용 시: **Cloud Functions(서버측)**에서 OCR·LLM 호출. 클라이언트 SDK는 Functions만 호출.
- 이미지·독후감은 동의받은 범위에서 처리, 외부 학습 비전송(약관 확인), 보관·파기 정책 명시.

---

## 2. OCR (한글 손글씨) — `ocrModule` 교체
손글씨, 특히 초등학생 글씨라 **정확도가 관건**(구현기술 안정성 점수).

| 후보 | 강점 | 유의 |
|---|---|---|
| **네이버 CLOVA OCR** | 한글·손글씨 인식 강점, 국내 서비스 | 유료(건당 과금), NCP 가입·도메인 등록 필요. 무료 크레딧 여부 확인 |
| **Google Cloud Vision** | 손글씨(DOCUMENT_TEXT_DETECTION), 다국어 | 월 무료 한도 후 과금. 한글 손글씨는 또박체에 유리 |
| (대안) 온디바이스 | 비용 0·오프라인 | 한글 손글씨 정확도 낮음 → 비권장 |

### ★자체호스팅 무료 OCR (현재 채택) — 비용 0 + 개인정보 안전 + 대회 가점
클라우드 API(CLOVA/Google) 대신 **오픈소스 OCR을 우리 쪽에서 구동**. 이유: ① 지속 비용 0 ② 학생 사진이 외부로 안 나감(개인정보 안전) ③ 직접 구축·파인튜닝=구현기술·독창성 점수.

| 엔진 | 위치 | 비고 |
|---|---|---|
| **Tesseract.js** | 브라우저(WASM) | 서버·비용 0, 현재 프로토타입에 **실연결 완료**(`ocrUpload`). 손글씨 약함→원고지/또박·인쇄 위주 |
| **EasyOCR** | Python 백엔드 | 한글·손글씨 Tesseract보다↑. Oracle Cloud Free Tier(평생무료 ARM)·HF Spaces 무료 호스팅 |
| 파인튜닝 | — | **AI Hub 한글 손글씨/필기체 OCR 데이터(무료)**로 EasyOCR 추가학습 시 비약적 상승 |

**실측(2026-06-29)**: Tesseract.js로 실제 학생 줄공책 손글씨(라면/숲) 인식 → 또박또박 쓴 첫 줄은 반쯤 인식("라면을 먹으면 숲이 사라질까"≈), 흘림 부분은 깨짐, **평균 신뢰도 32%**. → **무료OCR+흘림손글씨=사용곤란** 확인.

**되게 만드는 3원칙**: ① **원고지·또박체 입력 유도**(칸 분리=인식↑, 원고지 모드 이미 구현) ② **EasyOCR(+AI Hub 손글씨 파인튜닝)** ③ **학생이 OCR 결과 직접 수정**(사람-확인 루프=[6국03-05], 정확도 부담 제거).

**로드맵**: (지금)Tesseract.js 브라우저 PoC → (정확도 필요시)EasyOCR Python 백엔드(Oracle Free Tier)로 `ocrModule` 교체 → (고정밀)AI Hub 데이터 파인튜닝.

**참고(클라우드 대안, 비채택)**: CLOVA OCR(한글 손글씨 강점, 유료), Google Vision(무료한도 후 과금). 단 학생 사진 외부전송 이슈로 학교 서비스엔 자체호스팅이 우위.

### 교체 인터페이스
```js
// 프로토타입: 내장 샘플 반환
async function ocrModule(image){ /* 데모 */ }
// 실서비스: 서버 함수 호출
async function ocrModule(file){
  const fd=new FormData(); fd.append('image', file);
  const res=await fetch('/api/ocr',{method:'POST',body:fd});
  const {text}=await res.json();   // 서버가 CLOVA/Vision 호출 후 텍스트 반환
  return {book:'', raw:text};
}
```
- 서버 `/api/ocr`: 이미지 수신 → OCR API 호출 → 줄바꿈 정리 → 텍스트 반환.
- 전처리 팁: 회색조·대비 보정·기울기 보정으로 인식률↑.

## 3. AI 첨삭(LLM) — `aiReview` 교체
독후감 텍스트 → 성취수준 루브릭 기반 칭찬/개선/발전.

| 후보 | 비고 |
|---|---|
| **OpenAI GPT** | 한국어 첨삭 품질 우수, API 키 필요 |
| **Google Gemini** | 무료 한도 비교적 넉넉(확인 필요), 멀티모달(이미지+텍스트 → OCR까지 한 번에 가능) |
| (국내) HyperCLOVA X 등 | 국내 데이터·약관 측면 검토 가치 |

**권장**: Gemini 또는 GPT 중 **무료/저비용 한도 + 한국어 품질**로 선택. Gemini는 **이미지를 직접 넣어 OCR+첨삭 동시**도 실험 가치.

### 시스템 프롬프트(루브릭 주입) 예시
```
너는 초등 5~6학년 독후감을 첨삭하는 친절한 선생님이다.
학생 글을 읽고 2022 개정 국어 성취수준에 따라 평가하라.
반드시 JSON으로만 답하라:
{ "level":"A|B|C",
  "praise":["칭찬1","칭찬2"],
  "fix":["개선1(맞춤법·문장)","개선2"],
  "grow":["발전1(삶과 연결·사고 확장, 성취기준 코드 포함)","발전2"] }
기준:
- A: 줄거리 파악 + 풍부한 감상 + 삶과 적극 연관·성찰([6국05-06])
- B: 감상은 있으나 삶 연결 약함 → 그쪽을 발전 제안
- C: 줄거리 위주 → 느낌·생각을 발전 제안
- 어조: 격려 중심, 초등학생 눈높이. 맞춤법 지적은 가볍게.
```
### 교체 인터페이스
```js
// 실서비스
async function aiReview(text, book){
  const res=await fetch('/api/review',{method:'POST',
    headers:{'Content-Type':'application/json'},
    body:JSON.stringify({text, book})});
  return await res.json();  // {level, praise[], fix[], grow[], pts}
}
```
- 서버 `/api/review`: 위 시스템 프롬프트 + 학생 글 → LLM → JSON 파싱 → 포인트 계산(A30/B20/C10) 후 반환.
- **안전장치**: 출력 JSON 검증, 부적절어 필터, 실패 시 규칙기반 폴백(현 목업 재사용).

## 4. 키 발급 체크리스트(사용자 액션)
- [ ] **CLOVA OCR**: 네이버클라우드플랫폼(NCP) 가입 → CLOVA OCR 신청 → 도메인/시크릿 발급.
- [ ] **Google Vision/Gemini**: Google Cloud 프로젝트 → API 사용 설정 → API 키.
- [ ] **OpenAI**(선택): platform.openai.com → API 키.
- [ ] 각 서비스 **무료 한도·과금·교육용 약관·연령정책** 확인(스크린샷 보관 = 저작권/약관 증빙).
- [ ] 키는 서버 환경변수에만 저장, 깃 커밋 금지.

## 5. 비용·규정 메모
- 학급 20~30명 × 주 1편 = OCR·LLM 호출 월 ~100건 수준 → 무료~소액 범위 설계 가능(정확 한도는 발급 시 확인).
- **연령제한**: 학생 계정이 외부 AI에 직접 접속하지 않음(서버 경유)으로 약관 위반 회피 — 보고서에 명시(차별·안전 포인트).
- **개인정보**: 독후감·이미지 처리 동의(서식9), 외부 학습 비전송 옵션 확인, 보관기간·파기.
- **저작권**: 폰트·아이콘·이모지·도서표지 = 라이선스 확인. (현 프로토타입은 시스템 이모지 사용)

## 6. 전환 순서(PoC → 베타)
1. `/api/ocr` 1개 엔드포인트로 손글씨 1장 텍스트화(가장 큰 리스크 선검증).
2. `/api/review`로 루브릭 첨삭 JSON 1건 성공.
3. 프로토타입의 `ocrModule`/`aiReview`만 교체 → 나머지 UI·데이터·게이미피케이션 그대로.
4. Firebase Auth/Firestore로 localStorage 대체(다학교·영속).
