# 「책갈피」 개발 보고서
### 손글씨 OCR + AI 첨삭 기반 독서활동 종합 플랫폼

> 제20회 정보화연구대회(디지털교육연구대회) · 교육용 SW·AI 분과 출품작
> 작성일: 2026-06-29 · 프로토타입 v0.7 기준

---

## 1. 개요

### 1.1 개발 배경
독후감 지도는 교사에게 높은 첨삭 부담을 주고, 학생에게는 "무엇을·어떻게 써야 하는지" 막막함을 준다. 기존 우수작(맞춤법 상담소 등)은 **맞춤법 교정**에 강점이 있으나, 독후감의 본질인 **내용 이해·감상·사고의 확장**을 평가·피드백하지 못한다.

「책갈피」는 ① 종이에 쓴 손글씨 독후감을 **사진 OCR**로 디지털화하고, ② **2022 개정 국어과 성취수준 5축 루브릭**으로 AI가 첨삭하며, ③ 교사가 AI 초안을 **검토·확정**하여 업무를 경감하고, ④ 포인트·캐릭터·랭킹으로 **읽기 동기**를 부여하는 독서활동 종합 플랫폼이다.

### 1.2 핵심 차별점
| 구분 | 기존 도구 | 책갈피 |
|---|---|---|
| 평가 대상 | 맞춤법·표현 위주 | **내용 이해·감상·삶과 연결·구성·맞춤법(5축)** |
| 평가 체계 | 단순 교정 | **2022 개정 성취수준 루브릭(A/B/C) + 성취기준 코드** |
| 교사 역할 | 전량 수기 첨삭 | **AI 초안 → 교사 조정·확정**(업무 경감) |
| 손글씨 | 미지원 | **사진 OCR**(제미나이 비전 / 브라우저 OCR) |
| 동기 부여 | 약함 | 포인트·성장단계·캐릭터·학급/전국 랭킹·명예의 전당 |

## 2. 개발 환경 및 기술 구조

### 2.1 기술 스택
| 영역 | 사용 기술 | 비고 |
|---|---|---|
| 구조 | 단일 HTML + 바닐라 JavaScript | 빌드 도구 없음, 서버 없음(정적 배포 가능) |
| 데이터 | 브라우저 `localStorage` | 키 `chaek_db2`, 교체형 목업 → 실서비스 시 서버 연동 |
| 손글씨 OCR | Tesseract.js v5(CDN) / **Gemini 2.5 Flash 비전** | 키 있으면 제미나이 우선, 없으면 브라우저 OCR |
| AI 첨삭·진위 | Gemini 2.5 Flash (BYOK) | 사용자 무료 키를 기기에만 저장(서버 없음) |
| 폰트·UI | Pretendard, SVG 레이어 합성 캐릭터 | 색 토큰 시스템(네이비·스카이·민트·앰버) |
| 학교 데이터 | `schools.js`(전국 NEIS 학교) | 실제 학교 검색 로그인 |

### 2.2 설계 원칙
- **교체형 모듈(read-or-fake)**: OCR·AI 첨삭은 동일 인터페이스의 목업/실모듈로 교체 가능하게 설계(03 API연동 설계서 참조).
- **BYOK(Bring Your Own Key)**: AI 키는 서버를 두지 않고 사용자 기기에만 저장 → 개인정보·비용 부담 최소화.
- **단일 파일·무빌드**: 교육 현장 배포 용이성, 유지보수 단순화.

## 3. 주요 기능

### 3.1 인증·학급 관리
- 전국 실제 학교(NEIS) 검색 기반 로그인/회원가입(학생·교사)
- 같은 학교·학년·반으로 가입 시 자동으로 같은 학급 편성
- 교사의 학생 **비밀번호 초기화**, 학생 추가/삭제

### 3.2 학생 화면
| 기능 | 내용 |
|---|---|
| **내 서재(홈)** | 캐릭터·레벨/칭호·**마스터 도달률**, 성장 그래프(스파크라인)+독서 나무, 뱃지 |
| **책 발견 캐러셀** | 학년 맞춤·이달의 추천·인기 독후활동·**대출 Top 10**(가로 캐러셀) |
| **독후감 쓰기** | 키보드 / **원고지(200칸 그리드)** / **사진 OCR** 3가지 입력 |
| **AI 첨삭** | 5축 루브릭 진단 + 칭찬·개선·발전 피드백 + **맞춤법 즉시 고치기** |
| **접근성 입력** | 음성 입력(STT)·읽어주기(TTS)·**쉬운 말 모드**(다문화·느린 학습자) |
| **고쳐쓰기** | 제출 글을 다시 다듬어 **향상도(이전 평균 대비)** 측정 |
| **단계 학습** | 6단계 성장 경로(씨앗→열매) + 5단계 작성 위저드(성취기준 연계) |
| **독서 게임** | 골든벨·밸런스 게임·빙고·대결 등(포인트 획득) |
| **나눔터** | 우수작 게시판(응원)·독서 토론방 |
| **상점(사물함)** | 아바타 6슬롯 장착(헤어·모자·상의·하의·신발·액세서리), 미리보기·구매·착용 |
| **랭킹·대회** | 우리 반·학교 내 학급·전국 학교(**포디움 Top 3**)·챌린지·명예의 전당 |

### 3.3 교사 화면 (관리 콘솔)
좌측 **사이드바 내비**(수업 진행 / 학생 관리 / 수업 설계 / 현황·명예)로 "관리 콘솔" 형태.

| 기능 | 내용 |
|---|---|
| **학급 대시보드** | 총 독후감·검토 대기·평균 포인트·A수준 비율(성장형 stat 카드), 성취수준 분포 |
| **학급 분석 인사이트** | 5축 분석으로 **약점 진단 + 추천 활동 + 성취기준** 자동 제시 |
| **학생별 현황** | **+포인트 지급**·격려 메시지·표창장(PDF)·가정통신문(PDF) |
| **독후감 검토** | AI 초안 → 교사 **5축 점수 조정 + 코멘트 확정**, 일괄 승인 |
| **진위 점검** | AI가 베끼기·AI생성 의심 탐지(보조 지표) |
| **손글씨 원본 열람** | 학생이 제출한 **사진 원본을 교사가 대조 열람**(📷 원본) |
| **미션·교내 대회** | 도서·기간 지정 미션 개설, 완료 현황 자동 집계 |

### 3.4 평가 체계 (교육과정 연계)
2022 개정 국어과 성취수준 기반 **5축 루브릭**:

| 축 | 의미 | 성취기준 |
|---|---|---|
| 내용 이해 | 인물·사건·배경 파악 | 6국05-03 |
| 감상 표현 | 생각·느낌 표현 | 6국03-03 |
| 삶과 연결 | 삶·경험과 연관·성찰 | 6국05-06 |
| 구성·근거 | 고쳐 쓰기·문장 연결 | 6국03-05 |
| 맞춤법 | 표기 정확성 | 6국04-06 |

→ 5축 평균으로 성취수준 **A/B/C** 산출, 교사가 최종 조정·확정.

## 4. 최근 고도화 작업 (2026-06 UI/기능 개선)

레퍼런스(정보이음·독서로·경북독서친구) 벤치마크 및 `frontend-design`·`web-design-guidelines`(skills.sh) 전문 가이드 적용.

| # | 작업 | 핵심 내용 |
|---|---|---|
| 1 | **교사 사이드바 콘솔 + 포디움** | 상단 탭 → 그룹형 좌측 사이드바, 랭킹 1·2·3위를 금·은·동 단상으로 |
| 2 | **접근성 감리** | `prefers-reduced-motion`·`:focus-visible` 링·`aria-current`·`tabular-nums` 보강 |
| 3 | **홈 책 발견 캐러셀** | 학년 맞춤·이달의 추천·인기 독후활동·대출 Top 10(4단 가로 캐러셀) |
| 4 | **칭호·도달률·+포인트** | 독서 칭호 chip(책읽기 새내기→책갈피 마스터), 마스터 도달률 바, 교사 수동 포인트 지급 |
| 5 | **인기도서 보강** | 깜냥·아몬드·기억 전달자·회색 인간 등 실제 인기작 추가 |
| 6 | **명예의 전당 히어로 카드** | 학급 총 포인트·마스터 수·현재 1위 실시간 배너 |
| 7 | **성장형 stat 카드** | 밋밋한 숫자 카드를 아이콘·색 코딩·맥락 정보 카드로 차별화 |
| 8 | **아바타 사물함/인벤토리** | 캐릭터 + 6개 장착 슬롯 + 인벤토리 그리드(게임 로커 느낌) |
| 9 | **제미나이 비전 OCR + 사진 원본 교사 열람** | 키 있으면 AI가 손글씨 정밀 인식, 제출 사진을 교사가 대조 열람 |

## 5. 검증

- **기능 회귀 검증**: 브라우저 DOM·레이아웃 계측으로 각 기능 동작 확인(사이드바 정렬·포디움 배치·캐러셀 스크롤·사물함 좌우 정렬·교사 사진 열람 체인 등).
- **콘솔 오류 0건** 유지.
- **접근성**: 모션 민감 사용자 대응, 키보드 포커스 가시화, 스크린리더용 시맨틱(`<nav>`/`aria-*`/`alt`).
- **반응형**: 데스크톱(사이드바·2단 레이아웃) ↔ 모바일(스택·가로 스크롤) 전환 확인.
